## Project 2: AI-Powered EHR Assistant
**File:** `Project 2 - AI Health Assistant - Jason McGinty.ipynb`

**Description:**
Development of an AI-powered Electronic Health Record (EHR) assistant designed to intelligently interpret medical data and assist with specific patient cases. 

**Key Outcomes & Methodologies:**
* **Validation & Safety:** Emphasized the necessity of rigorous testing and explicit safeguards to prevent incorrect medical guidance, including clear protocols for emergency situations (e.g., directing patients to 911).
* **Architecture:** Proposed transitioning to a multi-agent processing framework involving a dedicated validation agent to ensure higher accuracy and mitigate medical or legal exposure.
* **Compliance:** Incorporated strategies for engaging legal teams to ensure the intelligent agent aligns with strict regional healthcare regulatory requirements.

## Project 3: AI Delivery Exception Automation
**File:** `Project 3 - AI Delivery Exception Automation - Jason McGinty.ipynb`

**Description:**
A multi-agent Python application designed to autonomously handle and automate delivery exceptions processing using CSV files for data ingestion.

**Key Outcomes & Methodologies:**
* **Multi-Agent System:** Engineered a complex orchestrator using LangChain and LangGraph alongside large language models to resolve logistical issues without human intervention where possible.
* **Security Integration:** Implemented robust security measures, including comprehensive prompt-injection detection to filter out jailbreak attempts, HTML injections, and role-hijacking.
* **Business Logic Optimization:** Formulated strategic business recommendations to prioritize VIP/Premium delivery cases, implement human-in-the-loop safeguards for edge cases, and utilize GPU parallelization for faster performance.
